# ValueLens 開發與錯誤追蹤日誌 (Dev Log)

這份文件記錄了開發過程中遇到的錯誤、邏輯失誤及其修復方案，作為未來修改的警戒與參考。

---

## 🗓️ 2026-04-30

### 1. 全白畫面錯誤 (White Screen Bug)
*   **預計情況**：使用 `cv2.LUT` 加速渲染，將背景淡化 (Edge Mix) 預先算入查表。
*   **改錯的東西**：忽略了 `edge_mix` 應該只在 `edge_enabled=True` 時套用。當 Mix=100 且邊緣關閉時，LUT 被強行刷白。
*   **實際提升**：修正後畫面恢復正常，且渲染速度維持在 < 5ms。
*   **未來預防**：修改渲染邏輯時，必須檢查所有影響色彩的 UI 參數（Mix, Alpha 等）在「關閉狀態」下的行為。

### 2. 殘留屬性引用錯誤 (AttributeError: super_dense_detector)
*   **預計情況**：簡化偵測器架構，移除 redundant 的網格偵測。
*   **改錯的東西**：只刪除了初始化，卻漏掉了 `on_auto_balance_target_requested` 內部的呼叫。
*   **實際提升**：統一使用 `dense_detector` (隨機採樣)，代碼更簡潔。
*   **未來預防**：刪除任何屬性或方法前，必須進行全域 `grep` 搜尋所有引用點。

### 3. 定時器命名與邏輯混亂
*   **預計情況**：將主掃描頻率提升至 60Hz。
*   **改錯的東西**：在 `__init__` 中引用了未定義的 `self.refresh_timer`，且與 `self.timer` 功能重疊，導致潛在崩潰與邏輯冗餘。
*   **實際提升**：統一使用 `self.timer` 並連結至 `refresh_frame`，成功達成穩定 60 FPS 掃描。
*   **未來預防**：在修改 `__init__` 等核心初始化區塊時，優先檢視現有成員變數名稱，避免重複建立功能相似的對象。

### 4. 預設集索引溢出 (IndexError: list assignment)
*   **預計情況**：擴充預設集 Slot 從 20 到 40。
*   **改錯的東西**：忽略了磁碟上的 `settings.json` 會覆蓋 `field` 的預設值。
*   **實際提升**：在 `SettingsManager` 加入自動補齊邏輯，確保版本相容。
*   **未來預防**：涉及資料結構擴充（List, Dict）時，必須在 `load` 階段實作「補齊 (Filling)」機制。

### 5. 測試污染生產設定 (Test Pollution)
*   **預計情況**：進行全功能壓力測試。
*   **改錯的東西**：測試腳本直接使用實體的 `AppStore` 並觸發了 `save()`，導致使用者的階層數被隨機改成了 7 階。
*   **實際提升**：在測試腳本中將 `store.save` 替換為空函式 (Mock)，防止設定被篡改。
*   **未來預防**：自動化測試腳本應始終隔離 IO 操作（存檔、刪除），除非該測試的目的就是測試 IO。

### 6. NumPy 類型序列化錯誤 (TypeError: int32 is not JSON serializable)
*   **預計情況**：自動平衡運算後，將結果存入設定檔。
*   **改錯的東西**：運算結果為 `np.int32`，原生 `json` 模組不支援。
*   **實際提升**：實作 `SettingsEncoder` 自動轉換所有 NumPy 類型。
*   **未來預防**：在處理含有科學運算的資料結構存儲時，必須考慮到資料類型的轉換問題。
